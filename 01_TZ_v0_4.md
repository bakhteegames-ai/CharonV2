# ТЗ — Charon Canon v0.4

## 1. Цель
Собрать publishable web endless runner на одном движковом skeleton с двумя вариантами контента (v2 / v3), пригодный для дальнейшей публикации и масштабирования на следующие BTG-проекты.

## 2. Технологический стек проекта
- Vite
- React
- TypeScript
- Three.js
- WebAudio
- runtime assets через `/public`

## 3. Платформы
- Desktop web
- Mobile web
- primary target: Yandex Games
- secondary potential target: Google Play wrapper / web build reuse

## 4. Обязательный MVP-канон

### 4.1 Session loop
- menu
- playing
- paused
- game over
- restart / resume

### 4.2 Input
Desktop:
- left/right: lane change
- down: crouch/slide
- up: jump only for v3
- Esc/P: pause

Mobile:
- swipe left/right
- swipe down
- swipe up only for v3
- pause button in UI

### 4.3 Obstacles and items
- BLOCK
- BARRICADE
- OVERHEAD
- COIN
- SHIELD

### 4.4 Content generation
- pattern/chunk driven
- difficulty waves
- capability gating
- safe vs risk coin routes

## 5. Current architectural requirement
Различия между версиями должны жить не в копипасте world/system logic, а в:
- `variant.ts`
- `ContentPacks.ts`
- `GameProfile.ts`
- visual/audio directives
- future platform adapter

## 6. Runtime asset requirements
Система уже умеет:
- загружать GLTF игрока через `VisualPlayer`
- включать fallback на primitive collider
- подхватывать audio urls через `AudioManager.applyAudioPack`
- применять environment theme через `VisualTheme`

Следовательно все новые реальные ассеты должны поставляться в формате, совместимом с этим pipeline.

## 7. Ограничения
- не ломать gameplay канон ради арта
- не делать тяжёлые mobile-hostile решения без причины
- не смешивать source assets и runtime assets
- не оставлять “магические” assets без documented target path
